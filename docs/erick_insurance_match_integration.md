# Erick Insurance Match Skill Integration

This branch adds Erick's component for the prior authorization capstone.

## What was added

- `skills/insurance_match_skill/`
- Insurance policy matching for plans A, B, C, and D
- Simulated payer-criteria evaluation
- Missing documentation detection
- Denial-risk scoring
- PDF compatibility with PDFs generated by `clinic_note_skill`

## Compatible workflow

```text
clinic_note_skill -> outputs/patient_<PatientID>.pdf -> insurance_match_skill -> denial risk
```

## New tools exposed to the ADK agent

- `evaluate_patient_from_csv(patient_id, csv_path="diabetes_updated.csv")`
- `evaluate_prior_authorization(note_text)`
- `evaluate_prior_authorization_pdf(pdf_path)`
- `list_policies()`
- `get_policy(policy_id)`

## Dataset note

The dataset includes Glucose, BMI, Outcome, Age, and Insurance, but not A1C.
Therefore, this skill uses Glucose as the primary objective lab variable and
flags A1C as missing documentation when relevant.

## Example prompts

```text
Evaluate patient 93810 from diabetes_updated.csv and return policy match, passed
criteria, failed criteria, missing information, and denial risk.
```

```text
Evaluate the generated prior authorization PDF outputs/patient_93810.pdf and
return the simulated insurance denial risk.
```
